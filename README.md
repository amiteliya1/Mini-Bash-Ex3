# פרויקט Mini-Bash Shell - תרגיל 3

## תיאור הפרויקט
בפרויקט זה מימשתי מפרש פקודות (Shell) בסיסי בשפת C, הפועל בסביבת Linux/WSL. התוכנית מאפשרת למשתמש להריץ פקודות חיצוניות ופנימיות תוך ניהול תהליכים ישיר מול ה-Kernel.

## לוגיקת התוכנית (Shell Loop)
התוכנית פועלת בלולאה אינסופית המורכבת מהשלבים הבאים:
1. **הצגת Prompt:** הדפסת `mini-bash$` למסך.
2. **קריאת קלט:** קבלת פקודה מהמשתמש באמצעות `fgets`.
3. **פירוק (Parsing):** חיתוך המחרוזת למילים (Tokens) באמצעות `strtok`.
4. **זיהוי פקודות:**
   - פקודות פנימיות (`exit`, `cd`) מבוצעות ישירות על ידי תהליך האב.
   - פקודות חיצוניות מורצות באמצעות יצירת תהליך חדש.

## קריאות מערכת (System Calls) בשימוש
במסגרת הפרויקט נעשה שימוש בקריאות המערכת הבאות כפי שנדרש בתרגיל:
* **`fork()`**: ליצירת תהליך בן (Child Process) שבו תרוץ הפקודה החיצונית.
* **`execvp()`**: להחלפת תוכן תהליך הבן בפקודה המבוקשת. הפונקציה מחפשת את הפקודה בנתיבים המוגדרים (כמו `/bin`) ובתיקיית הבית.
* **`waitpid()`**: גורם לתהליך ה-Shell (האב) להמתין עד לסיום ביצוע הפקודה על ידי הבן וקבלת סטטוס היציאה שלו.
* **`chdir()`**: משמש למימוש הפקודה הפנימית `cd` לשינוי תיקיית העבודה.
* **`fflush()`**: להבטחת הדפסה מיידית של ה-Prompt למסך.
